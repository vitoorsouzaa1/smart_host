## Database Schema

```
User
- id: String (PK)
- name: String
- email: String
- password: String (hashed)
- createdAt: DateTime
- updatedAt: DateTime

Property
- id: String (PK)
- title: String
- description: String
- price: Decimal
- address: String
- city: String
- state: String
- country: String
- zipCode: String
- latitude: Float
- longitude: Float
- bedroomCount: Int
- bathroomCount: Int
- maxGuestCount: Int
- propertyType: Enum
- createdAt: DateTime
- updatedAt: DateTime
- isActive: Boolean
- ownerId: String (FK to User)

PropertyImage
- id: String (PK)
- url: String
- caption: String
- propertyId: String (FK to Property)
- createdAt: DateTime

Amenity
- id: String (PK)
- name: String
- icon: String
- category: Enum

_AmenityToProperty (Junction Table)
- A: String (FK to Amenity)
- B: String (FK to Property)

Booking
- id: String (PK)
- userId: String (FK to User)
- propertyId: String (FK to Property)
- startDate: DateTime
- endDate: DateTime
- guestCount: Int
- totalPrice: Decimal
- status: Enum
- createdAt: DateTime
- updatedAt: DateTime

UnavailableDate
- id: String (PK)
- propertyId: String (FK to Property)
- date: DateTime
- reason: String

Review
- id: String (PK)
- userId: String (FK to User)
- propertyId: String (FK to Property)
- rating: Int
- comment: String
- createdAt: DateTime

Payment
- id: String (PK)
- bookingId: String (FK to Booking)
- amount: Decimal
- currency: String
- status: Enum
- paymentMethod: String
- createdAt: DateTime
```

This mockup document provides a visual representation of the main pages in the SmartHost rental application and the database schema that supports it. The actual implementation would involve creating React components with Next.js, styling with Tailwind CSS, and connecting to the PostgreSQL database using Prisma ORM.
